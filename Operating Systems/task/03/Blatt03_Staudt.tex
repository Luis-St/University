\section*{II}

\begin{enumerate}
	\item Mehrfachvererbung: Java unterstützt keine Mehrfachvererbung von Klassen.
	Wenn eine Klasse bereits von einer anderen Klasse erbt, kann sie nicht gleichzeitig von Thread erben, kann aber das Runnable-Interface implementieren.
	\item Separation of Concerns: Die Verwendung von Runnable ermöglicht eine bessere Trennung zwischen dem Verhalten des Threads (die run-Methode) und der Thread-Verwaltung selbst.
	\item Wiederverwendbarkeit: Ein Runnable-Objekt kann an mehrere Threads übergeben werden, wodurch dieselbe Aufgabe von verschiedenen Threads ausgeführt werden kann.
	\item Ressourcenfreundlicher: Die Implementierung von Runnable benötigt weniger Ressourcen, da nicht für jede Task eine vollständige Thread-Unterklasse erstellt werden muss.
	\item Thread-Pools: In modernen Java-Anwendungen mit Thread-Pools (z.B. via ExecutorService) werden typischerweise Runnable-Tasks verwendet.
\end{enumerate}

\section*{III}

Möglicher Programmablauf mit vermischten Druckjobs:

\begin{enumerate}
	\item Hänsel-Thread startet und beginnt mit dem Drucken von haensel.txt.
	\item Hänsel-Thread druckt \("\)haensel 1\("\).
	\item Der Scheduler unterbricht Hänsel und wechselt zum Gretel-Thread.
	\item Gretel-Thread beginnt mit dem Drucken von gretel.txt.
	\item Gretel-Thread druckt \("\)gretel 1\("\).
	\item Scheduler wechselt zurück zum Hänsel-Thread.
	\item Hänsel-Thread druckt \("\)haensel 2\("\) und \("\)haensel 3\("\).
	\item Scheduler wechselt zum Gretel-Thread.
	\item Gretel-Thread druckt \("\)gretel 2\("\) und \("\)gretel 3\("\).
\end{enumerate}

Mögliche Ausgabe:
haensel 1
gretel 1
haensel 2
haensel 3
gretel 2
gretel 3
