\section*{Aufgabe b)}

\subsection*{I) Beide Reisebüros verkaufen Ticket 3}
\begin{enumerate}
	\item Ausgangszustand: \texttt{VerfuegbareTickets = 4}
	\item Thread Reiseland: ruft \+texttt{TicketsVerfuegbar()} auf $\rightarrow$ \texttt{return true} (4 > 0)
	\item Thread Happy Travel: ruft \texttt{TicketsVerfuegbar()} auf $\rightarrow$ \texttt{return true} (4 > 0)
	\item Thread Reiseland: ruft \texttt{TicketVerkauf()} auf
	\begin{itemize}
		\item \texttt{nr = VerfuegbareTickets} $\rightarrow$ \texttt{nr = 4}
		\item Context Switch zu Happy Travel
	\end{itemize}
	\item Thread Happy Travel: ruft \texttt{TicketVerkauf()} auf
	\begin{itemize}
		\item \texttt{nr = VerfuegbareTickets} $\rightarrow$ \texttt{nr = 4} (noch unverändert!)
		\item \texttt{VerfuegbareTickets = 3}
		\item \texttt{return 4}
	\end{itemize}
	\item Thread Reiseland: setzt Ausführung fort
	\begin{itemize}
		\item \texttt{VerfuegbareTickets = 3} (überschreibt den Wert von Happy Travel)
		\item \texttt{return 4}
	\end{itemize}
	\item Beide Threads geben aus: "verkauft Ticket 4"
	\item \texttt{VerfuegbareTickets} ist jetzt 3, aber beide haben Ticket 4 verkauft
\end{enumerate}

\subsection*{II) Verkauf von Ticket 0 trotz synchronized}
\begin{enumerate}
	\item Ausgangszustand: \texttt{VerfuegbareTickets = 1}
	\item Thread Reiseland: ruft \texttt{synchronized TicketsVerfuegbar()} auf
	\begin{itemize}
		\item Erhält Monitor
		\item \texttt{return true} (1 > 0)
		\item Gibt Monitor frei
	\end{itemize}
	\item Thread Happy Travel: ruft \texttt{synchronized TicketsVerfuegbar()} auf
	\begin{itemize}
		\item Erhält Monitor
		\item \texttt{return true} (1 > 0)
		\item Gibt Monitor frei
	\end{itemize}
	\item Thread Reiseland: ruft \texttt{synchronized TicketVerkauf()} auf
	\begin{itemize}
		\item Erhält Monitor
		\item \texttt{nr = 1}
		\item \texttt{VerfuegbareTickets = 0}
		\item \texttt{return 1}
		\item Gibt Monitor frei
	\end{itemize}
	\item Thread Happy Travel: ruft \texttt{synchronized TicketVerkauf()} auf
	\begin{itemize}
		\item Erhält Monitor
		\item \texttt{nr = 0} (aktueller Wert!)
		\item \texttt{VerfuegbareTickets = -1}
		\item \texttt{return 0}
		\item Gibt Monitor frei
	\end{itemize}
	\item Ausgabe: \("\) Happy Travel verkauft Ticket \(0"\)
\end{enumerate}

Problem: Die Prüfung (TicketsVerfuegbar()) und die Aktion (TicketVerkauf()) sind zwar einzeln synchronisiert, aber nicht als atomare Einheit.
Zwischen der Prüfung und dem Verkauf kann ein anderer Thread dazwischenkommen.
